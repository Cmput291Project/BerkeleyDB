Use Case Number: 		6
Use Case Name: 			Review Taken Photos
Participating Actors: 	User

Goal: 		To allow the user to view any previously taken photos.

Trigger: 	The user selects a photo from a list of either the names of photos,
			or a photo gallery.

Precondition:	The User has previously taken some photos, or has some photos 
				inside their current /database/ to view.

Postcondition:	The User's display now shows a view of the photo they have 
			selected to open.

Basic Flow:
	1. The user selects a photo they would like to view from a list
	2. The photo is then displayed on their screen
	3. The user may hit the back button to exit the photo

Exceptions:
	1.0 There are no photos, then do nothing, the user shouldn't be able
		to select anything from an empty list.
